background :

Vox in Rama is a point-and-click interactive fiction game created by Bailey Foltz for her NYU Interactive Media Arts Capstone project.

In Vox in Rama, you play as a cat in 1650s London, trying to prevent the rat population from spreading a new wave of the plague while simultaneously trying to avoid the persecution of the witch hunt. The game is based on real historical events regarding the treatment of “familiars” during witch hunts in England, the corresponding decrease in the cat population as they were similarly linked to luciferianism, and the responding increase in the rat population which was theorized to be a significant factor in the emergence of the Great Plague of London in 1665. Vox in Rama is an interactive story created for entertainment, the lighthearted narrative approach allowing the user to take part in the storyline while also engaging them in a retelling of historically based events and demonstrating how unexpected factors might lead to significant outcomes. The game is available to play on itch.io.

process :

I began this game by developing the backstory, doing research on the historical aspects of my game and determining the central plot. Although the game is not a historical game, it is based on real life historical events.

I then tried to determine the core visual aspects as well as sketching out the user flow. Having the user control their own story through dialogue options was a core feature of the game.

I built Vox in Rama using the game engine Adventure Game Studio, an open source development tool built specifically for creating graphical point-and-click games. The viewport dimensions for this game are 540 x 405 pixels, and so all the visuals within the game are created in a pixelated art style reminiscent of many classic adventure games. All assets were drawn in Procreate and brought into Adventure Game Studio.

I was then able to use Adventure Game Studio’s scripting language which is based on the C programming language to add all of the functions to the game, incorporating action and dialogue into the gameplay.

•   The player must interact with the environment throughout the game, which is achieved by clicking certain areas or objects within a “room”.

•   The central aspect of gameplay is the dialogue the user has with other cats. The user is given options for different lines of dialogue which will branch off into different conversations depending on which option is chosen. While different dialogue options do not progress into completely different storylines, They do offer slightly different dialogue paths as one makes their way through the game.

In order to evaluate this game, I did two short rounds of playtests. The initial playtest was a supervised playthrough of a single level in order to see if the game was playable, while enjoyment of the game was a secondary concern. The second round of playtests was an unsupervised playthrough through itch.io followed by a survey that users had to fill out upon completion of the game. All testers were able to play the game all the way through, so this survey was based on one of my main goals upon beginning this project - how fun the game was.

I received a lot of feedback from my playtesters regarding changes they would like to see in a future version of the game, such as adjustments to the script to allow for more varied storylines, or the inclusion of sound in the form of background music and sound effects. I hope to keep working on these additions to be re-uploaded as game updates.
